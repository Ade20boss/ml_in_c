Phase 1 · ~12 months

Foundations — the bedrock you cannot skip

Without mastery of C, x86 assembly, and computer architecture, you will hit walls at every turn. This phase is non-negotiable. Most failed OS projects die here.

Core Language: C & Assembly

Book The C Programming Language

Kernighan & Ritchie. The original. Read it twice. Write every exercise. You need C at a deep, reflexive level — not just "I've used it."

Book C: A Reference Manual

Harbison & Steele. The definitive C reference. Use as a companion for undefined behavior, types, and the full language spec.

Book Computer Systems: A Programmer's Perspective

Bryant & O'Hallaron (CS:APP). Probably the single most important book on this list. Covers x86 assembly, memory hierarchy, linking, concurrency, and system-level I/O.

Video CS:APP Lab Videos — CMU

The official CMU course labs for CS:APP. Data lab, bomb lab, attack lab, malloc lab. Do all of them. csapp.cs.cmu.edu

Computer Architecture

Book Computer Organization and Design (RISC-V)

Patterson & Hennessy. The canonical architecture textbook. Understand datapaths, pipelining, caches, virtual memory from first principles.

Book Modern x86 Assembly Language Programming

Daniel Kusswurm. x86-64 assembly with SIMD, AVX, and system programming patterns. Essential for writing bootloaders and low-level kernel code.

Video Nand2Tetris — Coursera

Build a computer from logic gates up. Extraordinary for deeply understanding how hardware abstractions layer. Free on nand2tetris.org

Book The Art of 64-bit Assembly

Randall Hyde. Modern, comprehensive x86-64 assembly. Free online. artofasm.randallhyde.com

Phase 1 Milestones
1

Implement malloc/free from scratch

Explicit free list, coalescing, boundary tags. Pass all CS:APP malloc lab tests.

2

Write a working shell in C

fork(), exec(), pipes, signal handling, job control. This tests your UNIX system programming fluency.

3

x86 assembly bootloader

Write a bootloader in raw x86 assembly that boots, prints to screen, enters protected mode. BIOS or UEFI.

4

Disassemble and annotate a real binary

Take a small C program, disassemble it, annotate every instruction. Understand the ABI, stack frames, calling conventions.

Phase 2 · ~12–18 months

Kernel development — the heart of your OS

You will build a monolithic kernel targeting x86-64. Study existing kernels obsessively. Linux and xv6 are your teachers.

Essential Reading

Book Operating Systems: Three Easy Pieces

Arpaci-Dusseau (OSTEP). Free online. The best modern OS textbook. Virtualization, concurrency, persistence — covers everything at depth. ostep.org

Book Lions' Commentary on UNIX 6th Edition

John Lions. Classic source code walkthrough of UNIX. Deep understanding of a real OS from its creators' generation. Foundational.

Book Linux Kernel Development

Robert Love. The most readable tour of the Linux kernel internals. Process scheduler, memory management, VFS, networking. Read front-to-back.

Book Understanding the Linux Kernel

Bovet & Cesati. Deeper, denser. Every data structure, every algorithm. Use as reference alongside Love's book.

Book The Design of the UNIX Operating System

Maurice Bach. Timeless. Covers process model, file system, device drivers, IPC. The mental model here maps directly to Linux.

Book Modern Operating Systems

Andrew Tanenbaum. Rigorous theory with MINIX as reference. Strong on microkernel concepts, scheduling theory, distributed systems.

Project-Based Learning

Project xv6 — MIT's Teaching OS

A small, clean UNIX-like OS written in C for RISC-V and x86. Read every line. Then implement all the labs. Then extend it. MIT 6.1810

Project OSDev Wiki

The bible of OS development. Bootloaders, GDT, IDT, paging, keyboard, VGA, ACPI, SMP. Bookmark everything. wiki.osdev.org

Video Write Your Own OS — YouTube

Nanobyte channel. Outstanding series building an x86-64 OS from scratch. Bootloader → protected mode → long mode → kernel. Highly recommended.

Video MIT 6.1810 OS Engineering

Full course lectures on YouTube. Covers xv6 implementation in depth. Top-tier instruction from MIT professors. Pair with the labs.

Kernel Subsystems to Implement (in order)
1

Bootloader + protected/long mode

BIOS or UEFI, real → protected → 64-bit long mode, load kernel ELF, set up stack.

2

GDT, IDT, exception handling

Global Descriptor Table, Interrupt Descriptor Table, page faults, general protection faults, timer IRQ.

3

Physical memory manager

Bitmap or buddy allocator. Track free/used frames. Parse memory map from bootloader.

4

Virtual memory + paging

4-level page tables, kernel/user space split, page fault handler, demand paging.

5

Process & thread model

PCB/TCB, context switching in assembly, scheduler (start with round-robin, then CFS-like).

6

System call interface

SYSCALL/SYSRET, user/kernel privilege boundary, system call table, argument passing.

7

VFS + file systems

Virtual filesystem layer, then implement ext2 or FAT32, then your own FS.

8

Device drivers

PCI enumeration, keyboard (PS/2 + USB HID), NVMe/AHCI storage, network card (e1000).

Phase 3 · ~12–18 months

Full OS — userspace, networking, and security

The kernel is just the beginning. A real OS needs a userspace, libc, shell, networking stack, and security model. This phase makes it production-worthy.

Networking

Book TCP/IP Illustrated, Vol. 1 & 2

Richard Stevens. The definitive networking reference. Vol 1: protocols. Vol 2: the BSD implementation. You will implement a TCP/IP stack — this is your guide.

Book Computer Networks: A Systems Approach

Peterson & Davie. Excellent system-level networking book. Strong on implementation details. Free online. systemsapproach.org

Security & OS Hardening

Book The Art of Exploitation

Jon Erickson. Understand exploitation to defend against it. Stack/heap overflows, shellcode, return-oriented programming. Essential for hardening your kernel.

Book Secure Coding in C and C++

Robert Seacord. Every pitfall in systems programming. Integer overflows, format strings, race conditions. Read before you ship any kernel code.

Userspace & libc

Book The Linux Programming Interface

Michael Kerrisk. 1500 pages on Linux system programming. The reference for building a POSIX-compliant userspace on top of your kernel.

Project musl libc source

Read musl libc — a clean, readable POSIX C library. Model your own libc on it. Smaller and cleaner than glibc. musl.libc.org

Advanced Scheduler & Concurrency

Book The Art of Multiprocessor Programming

Herlihy & Shavit. Lock-free data structures, memory models, CAS operations. Essential for your SMP kernel with a scalable scheduler.

Book Is Parallel Programming Hard? (perfbook)

Paul McKenney. The RCU expert's free book on parallel programming at kernel scale. Memory barriers, locking, RCU. Free PDF

Phase 3 Milestones
1

TCP/IP stack from scratch

Ethernet, ARP, IPv4, ICMP, UDP, TCP with handshake, flow control, retransmission.

2

Port a minimal libc

Implement or port musl libc. Get printf, malloc, file I/O, pthreads working on your kernel.

3

Port bash or build a shell

Get a real shell running. This validates your system call interface and process model.

4

SMP support

APIC, multi-core boot, per-CPU data, SMP-safe locking, load balancing scheduler.

5

Security: capabilities + KASLR + SMEP/SMAP

Privilege model, kernel ASLR, supervisor mode execution/access prevention.

Phase 4 · ~12–18 months

Browser engine core — HTML, CSS, and rendering

A browser engine is one of the most complex software systems ever built. You will build a full rendering engine: HTML parser → DOM → style → layout → paint → composite.

Essential Theory

Book Web Browser Engineering

Pavel Panchekha & Chris Harrelson. Build a browser from scratch in Python, then translate to C/C++/Rust. The ONLY book specifically on browser internals. Free online. browser.engineering

Video "How Browsers Work" — Tali Garsiel

The legendary 2011 article, still the best overview of browser rendering pipeline internals. Read this first, read it again later. web.dev

Video Inside look at modern web browser — Google

4-part series by Google engineers on Chrome's multi-process architecture, compositor threading, and rendering pipeline. Essential mental model.

Project Servo — Mozilla's research browser

Read the Servo source (Rust). It's the cleanest modern browser engine codebase for learning. Parallel layout, WebRender. github.com/servo/servo

Specifications (must read)
  • HTML Living Standard — html.spec.whatwg.org — Parsing algorithm, tokenizer state machine, DOM construction
  • CSS Specifications — drafts.csswg.org — Cascading, specificity, box model, flexbox, grid
  • CSS Layout API / Houdini — Understanding the layout worklet API clarifies the box model deeply
  • Encoding Standard — Character encoding handling, UTF-8, BOM detection
  • URL Standard — URL parsing is surprisingly complex and spec-defined
  • Fetch Standard — HTTP fetching model, CORS, credentials
Rendering & Graphics

Book Real-Time Rendering, 4th Ed.

Akenine-Möller et al. GPU rendering pipeline, shading, compositing. Essential for your GPU-accelerated paint stage and compositor.

Book Computer Graphics: Principles and Practice

Hughes, van Dam et al. From rasterization to ray tracing. Understanding 2D rasterization is fundamental to text rendering, bezier curves, SVG.

Book Digital Typography — Donald Knuth

Text rendering is brutally hard — font metrics, hinting, subpixel rendering, kerning, ligatures, bidirectional text. Start with HarfBuzz and FreeType, but understand the theory.

Project Skia Graphics Library

Chrome's 2D graphics library. Study the architecture. You'll either use it, fork it, or build something similar. skia.org

Rendering Pipeline: Build Order
1

HTML tokenizer + tree construction

Implement the WHATWG tokenizer state machine exactly. It's 80+ states. Do not shortcut this.

2

CSS parser + cascade

Tokenize CSS, build CSSOM, compute specificity, apply cascade. Implement inheritance.

3

Style resolution

Attach computed styles to DOM nodes. Handle pseudo-classes, pseudo-elements, media queries.

4

Layout engine

Box model, block layout, inline layout, floats, flexbox. This is the hardest part of a browser engine.

5

Paint + rasterization

Walk the paint tree, issue draw commands, rasterize to pixels. Start with a CPU rasterizer.

6

GPU compositing

Layer promotion, tile-based rendering, GPU upload, OpenGL/Vulkan compositing. Chrome's approach is the reference.

Phase 5 · ~12–18 months

Full browser — JavaScript engine, networking, and security

A rendering engine without JavaScript is a PDF viewer. This phase adds a JS engine, the network stack, and browser security architecture.

JavaScript Engine

Book Crafting Interpreters

Robert Nystrom. Build a bytecode VM from scratch. Best introduction to language implementation. Free online. craftinginterpreters.com

Book Engineering a Compiler

Cooper & Torczon. Production-grade compiler theory. SSA form, register allocation, instruction selection. Necessary for a JIT compiler.

Book Writing an Interpreter/Compiler in Go

Thorsten Ball. Two-book series. Lexer → parser → AST → bytecode → VM. Extremely clear prose. Great warmup before tackling a JS engine.

Project V8, SpiderMonkey & JavaScriptCore source

Read and study all three engines. V8's Ignition + Turbofan, SpiderMonkey's WarpMonkey, JSC's DFG/FTL. The JS spec is ECMA-262. tc39.es

Book Virtual Machines — Smith & Nair

Process VMs, language runtimes, JIT compilation theory. Essential for understanding how a production JS engine works under the hood.

Video Franziska Hinkelmann — JS engines YouTube talks

V8 team engineer's talks on hidden classes, inline caches, and JIT compilation. Concrete, low-level, highly recommended.

Browser Networking

Book High Performance Browser Networking

Ilya Grigorik. HTTP/1.1, HTTP/2, HTTP/3, QUIC, WebSockets, WebRTC. Free online. The definitive guide to browser networking. hpbn.co

Book HTTP: The Definitive Guide

Gourley et al. Every HTTP concept: caching, authentication, HTTPS, connection management. Essential for your browser's network layer.

Browser Security

Book The Web Application Hacker's Handbook

XSS, CSRF, clickjacking, SQL injection — understand all attacks to build defenses. Your browser must defend against the entire web.

Book The Tangled Web

Michal Zalewski. The security model of the web — same-origin policy, content sniffing, MIME types, security headers. Written by a Google security researcher.

Phase 5 Milestones
1

Build a bytecode interpreter for ES5

Lexer → parser → AST → bytecode → VM. Get a basic JS subset running in your engine.

2

Add a JIT compiler

Profile the bytecode interpreter, identify hot functions, compile to native x86-64 machine code using a simple JIT. Study V8's Ignition/Turbofan split.

3

Garbage collector

Implement a generational GC. Young/old generation, minor/major GC, write barriers, card tables.

4

HTTP/2 client + TLS 1.3

HPACK header compression, stream multiplexing, TLS 1.3 with modern cipher suites.

5

Site isolation + sandboxing

Multi-process architecture, process-per-site, renderer sandbox. Study Chromium's architecture.

Phase 6 · Ongoing

Production — performance, tooling, and shipping

The gap between a working prototype and a production system is enormous. This phase covers the engineering discipline needed to ship something world-class.

Performance Engineering

Book Systems Performance, 2nd Ed.

Brendan Gregg. The bible of performance analysis. CPU, memory, I/O, network profiling. BPF, perf, flamegraphs. Use this to profile and optimize every layer of your OS and browser.

Book BPF Performance Tools

Brendan Gregg. eBPF for tracing and profiling at kernel level. Write BPF programs to analyze your OS's own behavior in production.

Book Every Computer Performance Book

Bob Wescott. Queuing theory, capacity planning, throughput vs. latency tradeoffs. Helps you design performant kernel subsystems analytically.

Book Structured Computer Organization

Andrew Tanenbaum. Cache coherency, NUMA architectures, memory hierarchies. Critical for squeezing performance from multi-socket hardware.

Build Systems, Testing & DevOps

Project WPT — Web Platform Tests

The official test suite for browser compatibility. Run WPT against your browser. Passing WPT is the bar for a real browser. web-platform-tests.org

Project POSIX conformance tests

LSB (Linux Standard Base) test suite. Run against your OS to validate POSIX compliance. Also run LTP (Linux Test Project) for kernel testing.

Advanced Topics to Master
  • WebAssembly (WASM) — Implement a WASM runtime in your browser engine. Spec: webassembly.github.io/spec
  • WebGPU / WebGL — GPU API surface in the browser. Implement the JS API layer over your Vulkan/OpenGL backend.
  • Accessibility tree (AX tree) — ARIA roles, accessibility tree construction. Non-negotiable for a real browser.
  • Internationalization — Unicode, bidirectional text (BIDI), complex scripts (Arabic, Devanagari). HarfBuzz + ICU.
  • Spectre/Meltdown mitigations — Site isolation, timer jitter, process model decisions are all driven by hardware vulnerabilities.
  • Power management (ACPI) — Sleep states, CPU frequency scaling, battery management for your OS.
  • Display server / Wayland — Implement a Wayland compositor or DRM/KMS direct rendering for your OS's GUI layer.
